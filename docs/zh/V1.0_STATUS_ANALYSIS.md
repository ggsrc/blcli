# blcli v1.0 能力实现状态分析

> 分析日期：2026-06-25（修订）  
> 基于：`docs/zh/Roadmap.md` Phase 1 vs 当前 main 分支代码

## 执行摘要

**总体完成度：约 95%（GCP-first v1 可发布）**

- ✅ **Phase 1 核心**：init / apply / status / rollback / explain / check / destroy
- ✅ **apply 按 project**：terraform、kubernetes、gitops 均已支持 `--project`
- ✅ **status 机器可读**：`--format=table|json|yaml`
- ✅ **Resume**：`init` 与 `apply all` 支持检测未完成操作并续跑（`--no-resume` 可跳过）
- ✅ **失败指引**：关键路径输出可操作的 next steps
- ❌ **非 v1 范围**：bootstrap 向导、配置市场、多云、workflow、monitor、Agent 三件套命令

| 能力 | 状态 | 说明 |
|------|------|------|
| 一键初始化所有模块 | ✅ | `blcli init` |

## Phase 2: 用户体验（不属于 v1 发布门槛）

以下项在 Roadmap Phase 2，**不应阻塞 v1 发布**：

| 能力 | v1 策略 |
|------|---------|
| 交互式配置向导 | v1.5+；personal 模板 `init-args --profile` 已覆盖部分场景 |
| `blcli bootstrap` session | Roadmap 标注 **暂不实现** |
| 操作时间估算 | 不做 |
| 配置模板库 / 市场 | v2 生态 |
| 独立审计日志 | v3；progress 文件为 v1 替代 |
| 失败自动重试 | 不做 |
| apply 失败自动 rollback | 不做 |

### Phase 2 中已以更小粒度落地的部分

| 能力 | 实现方式 |
|------|----------|
| 错误可操作提示 | `PrintFailureHints`（非完整 diagnose 子系统） |

|------|------|
